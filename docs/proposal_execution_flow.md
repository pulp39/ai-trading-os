# Proposal Execution Flow (Operational Memo)

This document describes the minimal operational procedure for applying a proposal using the institutional scripts.

The procedure verifies that the institutional proposal pipeline works correctly.

## Proposal ID Rule

All proposals must use the following identifier format:

`P-YYYYMMDD-NNN`

Example:

`P-20260310-001`

The same `proposal_id` must be used consistently across:

- proposal filename
- institution_update.json
- trace_event metadata
- proposal discussions

Draft proposals:

`proposals/draft/P-YYYYMMDD-NNN.md`

Accepted proposals:

`proposals/accepted/P-YYYYMMDD-NNN.md`

## Step 1 ? Create Draft Proposal

Create the draft proposal file:

`proposals/draft/P-YYYYMMDD-NNN.md`

The proposal should describe:

- purpose
- scope
- expected outcome

## Step 2 ? Create institution_update.json

Create:

`tmp/institution_update_P-YYYYMMDD-NNN.json`

Required fields:

- proposal_id
- proposal_status
- trace_event
- proposal_file
- git

`proposal_status` must be:

`accepted`

## Step 3 ? Validate the Update Payload

Run the validator:

`python scripts/institution/validate_institution_update.py --input tmp/institution_update_P-YYYYMMDD-NNN.json`

Expected result:

`institution_update.json validation OK`

## Step 4 ? Apply Accepted Proposal

Execute the institutional apply script:

`python scripts/institution/apply_accepted_proposal.py --input tmp/institution_update_P-YYYYMMDD-NNN.json`

This will:

- copy the proposal from `draft` to `accepted`
- record the `trace_event`
- create metadata artifacts

## Step 5 ? Verify Results

Check:

1. Accepted proposal exists:

`proposals/accepted/P-YYYYMMDD-NNN.md`

2. trace_event recorded in database.

## Step 6 ? Commit the Result

`git add proposals`
`git add tmp`

`git commit -m "institution: apply proposal P-YYYYMMDD-NNN"`
`git push origin main`

## Purpose of This Procedure

This process ensures that proposals are applied through a controlled institutional pipeline rather than manual modification.

It guarantees:

- traceable proposal history
- consistent proposal identifiers
- validated institutional updates
- recorded system evolution via trace_event

## Next Phase

After the pipeline is verified, proposals can be generated by AI Proposers (e.g., Claude) and processed through the same institutional workflow.
