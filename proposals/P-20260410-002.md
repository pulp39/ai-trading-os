## 1. 提案背景

現行制度における課題：

- AIモデル名のみでは人格単位としての識別が曖昧
- 現行役職配置がAIスキルセットと最適にマッチしていない
- 大規模開発・制度運用における役割分担が非効率

---

## 2. 提案内容

### (A) AiiD再定義

**新定義:**

```
AiiD = AI名(フレームワーク単位) + スレッド名 + 役職名 + Founder認証情報
```

**例:**

```
ChatGPT / P-20260410-PHASE10 / Proposer / FounderSigned
```

**運用上の明確化（Librarian最終決定）:**

① **スレッド名の定義と安定性**
Phase固定とする。Phase単位でAiiDを一意に識別。
Session・会話単位の変動は影響なし。

② **既存 trace_event 互換方針**
新定義は新規記録から適用（非遡及）。
既存 agent_id は変更せず、既存履歴の整合性を保持。

**将来的な改良:**

- 現状はチャットとアンカーによるFounder任命
- 将来はATOSダッシュボード上で役職登録・Founder認証を自動付与
- これによりAiiDの必要要件としてのFounder承認を制度的に統合
