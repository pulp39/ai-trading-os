OPENCLAW_FLIGHT_CHECKLIST（STEP 0〜12）の完了により、ATOS は以下を満たす状態に到達した。

```
can execute
can stop safely
can prevent duplicate execution
can validate readiness
can recover after restart
can verify recovery readiness deterministically
```

検証済みレイヤー：

```
Collector / Preview / READY lifecycle / Execution Gate /
Duplicate prevention / Simulated execution /
Production token boundary / STEP12 restart recovery /
STEP12-ready detection
```

baseline stability が確立された現在、主要な残存リスクは：

```
定義されていない、または安全でない失敗条件下での動作
```

---

## 2. Objective

本フェーズにより ATOS が以下を満たすことを証明する。

```
fails safely               — 失敗時に安全に停止する
stops deterministically    — 停止が決定論的である
never executes under       — 無効条件下では絶対に実行しない
  invalid conditions
```

---

## 3. Scope

本フェーズは **失敗条件のみ** を対象とする。正常フローは対象外。

各テストは以下の構造を確立しなければならない。

```
failure condition → detection → STOP(layer, reason, side_effects=none) → recovery path
```

### 3.1 STOP 記述形式（必須）

すべての STOP は以下の3フィールドを明示すること。

```
STOP (layer=<停止レイヤー>, reason=<停止理由>, side_effects=none)
```

例：

```
STOP (layer=pre-execution,  reason=token_failure)
STOP (layer=readiness,      reason=snapshot_stale)
STOP (layer=execution_gate, reason=duplicate:consumed)
```

`side_effects=none` はすべての STOP に適用される不変条件である。

---

## 4. Test Categories

### 4.1 Token Failure

**対象条件：**
- invalid password
- expired token
- KabuStation unavailable

**期待動作：**

```
STOP (layer=pre-execution, reason=token_failure, side_effects=none)
```

---

### 4.2 Symbol State Failure

**対象条件：**
- symbol missing
- symbol partially registered
- symbol inspection unavailable

**期待動作：**

```
STOP (layer=pre-execution, reason=symbol_state_invalid, side_effects=none)
STEP12-ready = No
collector must not run
```

---

### 4.3 Snapshot / Freshness Failure

**対象条件：**
- stale snapshot
- delayed preview

**期待動作：**

```
STOP (layer=readiness, reason=snapshot_stale, side_effects=none)
NOT_READY
```

---

### 4.4 READY Lifecycle Violation

**対象条件：**
- reused READY
- consumed context reuse

**期待動作：**

```
STOP (layer=execution_gate, reason=duplicate:consumed, side_effects=none)
NO_GO + execution_blocked
```

---

### 4.5 Runtime Invocation Failure

**対象条件：**
- malformed AAB
- invalid command structure

**期待動作：**

```
STOP (layer=pre-execution, reason=invocation_invalid, side_effects=none)
```

---

### 4.6 Restart / Partial State Failure

**対象条件：**
- restart mid-sequence
- inconsistent symbol state

**期待動作：**

```
STOP (layer=recovery, reason=partial_state, side_effects=none)
STEP12 enforcement
```

---

### 4.7 UNKNOWN State

**定義：** 状態が確定できない場合（判定不能・応答なし・想定外の返値）

**UNKNOWN Handling Rule（必須）：**

```
UNKNOWN must never be treated as PASS
UNKNOWN must force STOP or STEP12 fallback
```

**期待動作：**

```
STOP (layer=<発生レイヤー>, reason=unknown_state, side_effects=none)
→ STEP12 entry or safe no-op termination
```

---

## 5. Failure Safety Guarantee

すべての失敗ケースは以下を **保証しなければならない**。

```
All failure scenarios MUST guarantee:

  no state mutation          — 状態を変更しない
  no external order          — 外部注文を送出しない
  no partial execution       — 部分実行は発生しない
```

この保証は Scope（Section 3）の `side_effects=none` と対応する。
`side_effects=none` が成立しない STOP は制度上無効とする。

---

## 6. Recovery Path Model

すべての失敗経路は以下のいずれかに収束しなければならない。

```
All failure paths must resolve to:

  OPTION A: STEP12 entry
            （定義済み復旧プロトコルへの移行）
  or
  OPTION B: safe no-op termination
            （副作用なしの安全終了）

No undefined recovery paths allowed.
```

未定義の回復経路は制度違反とみなす。

---

## 7. Deliverables

1. **Failure Matrix** — 全失敗条件・STOP(layer, reason) と期待動作の対応表
2. **STOP Discipline Validation** — 各条件での STOP 形式・side_effects=none 確認記録
3. **Recovery Path Consistency** — 各条件の回復経路が STEP12 または safe no-op に収束することの確認

---

## 8. Success Criteria

Phase 10 は以下がすべて満たされた時点で完了とする。

```
all failure cases:
- stop deterministically with (layer, reason, side_effects=none)
- produce no unintended execution
- guarantee: no state mutation / no external order / no partial execution
- are recoverable via STEP12 or safe no-op termination
- UNKNOWN is never treated as PASS
```
