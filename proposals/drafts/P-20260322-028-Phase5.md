# P-20260322-028-Phase5
## Minimal Live Real Order Test Framework

## 1. Background

P-028 により execution capability は制度的に確立済み：

```text
execution_blocked
→ execution_gate_passed
→ authorization_granted
→ execution_recorded (simulation)
→ execution_recorded (real_order_dry_run)
→ external_order_allowed

System state：

Authorized and capable of execution, but has not executed

2. Objective

Phase 5 の目的：

外部注文 transmission の最小制度成立の検証

3. Core Design Problem
Authorization の不可逆化
外部 transmission の不確実性
実行後の system 安全制御
4. Core Concepts
4.1 authorization_consumed

定義：

外部APIに live real_order payload が送信された瞬間

特性：

不可逆
1回限り
成否非依存
4.2 transmission_unknown

定義：

transmission 結果が確定できない状態

追加仕様：

timeout 必須
timeout 値は task artifact に明示

行動制約：

新規 order 禁止
retry 禁止
cancel / status inquiry のみ許可
人間確認要求
4.3 post_submit_safety_lock

定義：

live transmission 後の強制安全ロック

明確化：

常時必須（例外なし）
成功・失敗・unknown 全ケースで発動
4.4 live_order_window_open

定義：

Registrar が明示的に開く live execution 許可時間窓

制約：

開始・終了時刻を明示
最大時間を task artifact に定義
window 外では execution 不可
5. State Transition
external_order_allowed
→ live_order_window_open
→ authorization_consumed
→ external_order_submitted
→ external_order_acknowledged
   | external_order_rejected
   | transmission_unknown
→ post_submit_safety_lock
→ execution_blocked
6. Minimal Live Order Constraints
single symbol
single order
minimal quantity
single attempt
no retry
7. Authorization Model
authorization_granted
→ authorization_consumed
→ authorization_invalid

明示：

再利用不可
再試行不可
再実行には新規 authorization 必須（Phase 5 scope外）
8. Transmission Handling Policy
状態	許可される行動
acknowledged	cancel optional
rejected	fallback
unknown	cancel / inquiry only
any	new order prohibited
9. Rollback Definition
9.1 Institutional Rollback
execution_blocked
authorization 無効化
system freeze
9.2 Market Rollback

Phase 5に含める：

cancel flow を正式スコープに含める

cancel flow:

cancel request 送信
cancel 確認
trace_event 記録

条件：

未約定：必須
一部約定：残数量 cancel
unknown：cancel 試行許可
10. Phase 5 Invariants
I-01: 1 authorization = 1 transmission
I-02: authorization_consumed は不可逆
I-03: live execution は明示承認必須
I-04: execution 後は必ず blocked に戻る
I-05: block fallback は常時優先
I-06: retry 禁止
I-07: multi-order 禁止
I-08: window 外 execution 禁止
I-09: authorization 再利用禁止
I-10: cancel flow は bounded
I-11: transmission_unknown 中は新規実行禁止
I-12: post_submit_safety_lock は自動解除されない
11. Safety Principle

Execution must remain explicitly authorized and bounded

12. Clarifications
Scope外明示
authorization 再取得プロセス
継続的 live trading
strategy-based execution

不可逆性定義：明確
