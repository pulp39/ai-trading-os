# Anchor Governance Framework

## Purpose

This proposal establishes the governance framework for
anchor documents used within the AI Trading OS repository.

Anchor documents are operational context artifacts used to
share institutional state between AI agents. They are not
institutional records and do not constitute the Source of Truth.

---

# Institutional Documentation Layers

AI Trading OS documentation is organized into three layers.

## Layer A — Institutional Records

Authoritative system records.

Examples:

- constitution.md  
- proposals/*  
- founder_records/*  
- research.trace_event  

Layer A documents constitute the Source of Truth.

Modification requires institutional procedures defined
by the constitution and governance framework.

---

## Layer B — Institutional State Snapshot

Operational documents summarizing the current
institutional state for AI context sharing.

Example:

docs/anchors/AI_TRADING_OS_MASTER_ANCHOR.md

These documents are derived from Layer A records but are
not authoritative.

AI agents may reference Layer B for context but must
treat Layer A as the Source of Truth.

---

## Layer C — AI Operational Context

AI-specific operational runtime guidance.

Example:

CLAUDE.md

These documents provide behavioral context for specific
AI agents and are independent from institutional records.

---

# Update Authority

## Layer B

AI proposal → Librarian approval

Reason:
Institutional state summaries must remain consistent with
Layer A records.

---

## Layer C

AI proposal → Founder approval

Reason:
Operational instructions for AI agents fall under
Founder authority.

---

# Update Triggers

Primary triggers:

- CRC completion
- Constitutional amendment
- Founder institutional decisions
- Major governance events

Secondary triggers:

- Repository structure changes
- AI architecture changes

Anchors should change rarely to maintain context stability.

---

# Draft Anchor Handling

Draft anchors follow this process:

draft  
→ proposal artifact  
→ Librarian review  
→ Founder approval  
→ adoption or discard

Example draft:

AI_TRADING_OS_MASTER_ANCHOR_NEW_DRAFT_20260312.md

---

# Limitations

This proposal defines operational governance rules only.

It does not modify:

- constitution
- existing anchor content

Future updates to anchors follow the procedures defined here.
