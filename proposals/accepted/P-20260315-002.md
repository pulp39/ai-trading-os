# P-20260315-002
## Documentation Architecture Formalization After Anchor Restructuring

---

## Summary

This proposal formalizes the documentation architecture established
after the Phase 3 documentation restructuring. The restructuring
introduced a single documentation entry point, clarified document
layers, preserved legacy materials through archive relocation, and
reduced bootstrap ambiguity for AI participants.

---

## Objective

To formally recognize the current AI Trading OS documentation structure
as the active documentation architecture of the project.

---

## Rationale

Recent restructuring work established the following improvements:

- `docs/DOCS_MASTER_INDEX.md` now serves as the documentation entry map.
- `docs/anchors/ATOS_BOOTSTRAP_ANCHOR.md` remains the required
  institutional reconstruction starting point.
- `docs/ai_participant_onboarding.md` now explicitly points AI
  participants to the bootstrap anchor before role-specific onboarding.
- Legacy documents that no longer define current institutional state
  were preserved under `docs/archive/` rather than removed.

The resulting structure now clearly distinguishes:

- constitutional foundation
- anchor layer
- operational documentation
- ledger material
- archived historical material

This reduces the risk of partial institutional reconstruction and makes
the repository more legible to both human and AI participants.

---

## Requested Institutional Recognition

The following points are recognized as the current documentation
architecture of AI Trading OS:

### 1. Single Documentation Entry

`docs/DOCS_MASTER_INDEX.md` is recognized as the documentation entry
map.

### 2. Bootstrap Priority

All AI participants are expected to reconstruct institutional state
from:

```
docs/anchors/ATOS_BOOTSTRAP_ANCHOR.md
```

before relying on onboarding or role-specific operational documents.

### 3. Documentation Layer Model

The documentation structure is recognized as:

```
Constitution
↓
Anchors
↓
Operational Documents
↓
Ledgers
↓
Archive
```

### 4. Archive Preservation Principle

Legacy documents should normally be preserved through archival
reclassification rather than deletion when they no longer define
current institutional state.

### 5. Current Archive Categories

The archive structure introduced in Phase 3 is recognized as valid:

```
docs/archive/early_design/
docs/archive/deprecated_pipeline/
docs/archive/historical_notes/
```
