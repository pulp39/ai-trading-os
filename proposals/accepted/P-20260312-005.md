## Scope Statement

This proposal establishes operational governance rules for the classification,
update authority, and update triggers of anchor documents in AI Trading OS.

This proposal is not a constitutional amendment. Anchor documents are not
institutional records. This proposal clarifies their operational position.

## Documentation Layers

### Layer A — Institutional Records (Source of Truth)

- constitution.md
- proposals/*
- founder_records/*
- research.trace_event

### Layer B — Institutional State Snapshot

- docs/anchors/AI_TRADING_OS_MASTER_ANCHOR.md

Update authority: AI proposal -> Librarian approval

### Layer C — AI Operational Context

- CLAUDE.md

Update authority: AI proposal -> Founder approval

## Principles

- Anchors are not Source of Truth
- Anchors should change rarely
- AI agents must not autonomously modify any layer

## Draft Anchor Process

draft anchor -> proposal artifact -> review -> Founder approval
-> formal anchor update or discard record
