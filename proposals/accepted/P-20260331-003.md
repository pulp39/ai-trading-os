## 4. エラーハンドリング

### 4.1 発注失敗時の処理

```
if order_failed:
    trace_event: execution_failed
    readiness_state → NOT_READY（当該セッション内）
    Founder に通知
    自動リトライ禁止
```

### 4.2 部分約定・タイムアウト

- 部分約定: `execution_performed` に約定数量を記録し Founder に通知
- タイムアウト: 発注後 **30秒**（Founder Directive）で無応答の場合は `execution_failed` を記録
- タイムアウト超過時は即時 failure とし、**自動リトライ禁止**

## 8.1 Execution Kill Switch（Founder Directive）

Founder は以下の方法により、即時かつ確実に Execution を停止できる：

**実装方式: フラグファイル方式**

```
ファイルパス: scripts/openclaw/KILL_SWITCH.flag
```

- 上記ファイルが**存在する場合**、execution_gate.py は Gate 通過を拒否し即時停止
- ファイルの存在チェックは Gate 評価の**最優先**で実行される（他の全チェックより先）
- trace_event: `execution_gate_rejected`（reason: `kill_switch:active`）を記録

**Kill Switch 発動手順:**
```powershell
# 即時停止
New-Item -Path "scripts\openclaw\KILL_SWITCH.flag" -ItemType File

# 解除
Remove-Item "scripts\openclaw\KILL_SWITCH.flag"
```

**Kill Switch の制度的位置づけ:**
- Founder Directive として最上位の停止権限
- Proposal・Approval・Gate 条件に優先する
- 解除は Founder の明示的操作（ファイル削除）のみ
