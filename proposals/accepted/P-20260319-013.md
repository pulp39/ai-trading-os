Proposal P-20260319-013

Title: Execution Recording Framework
Date: 2026-03-19
From: Proposer (Drafted by Librarian)
Type: Execution Accountability Layer — Recording Definition
Constitutional Impact: None
Scope: 外部実行結果の制度的記録（最小構成）

Background

P-010〜012により、ATOSは以下を確立しました：

AAB（信頼）
↓
TMP（配送）
↓
AEEA（実行主体）

しかし現状では、

「何が実行されたか」
「何が拒否されたか」
「何が失敗したか」

を制度的に記録する仕組みが未定義です。

本Proposalはこれを最小構成で定義します。

Core Principle
すべての実行結果は記録されなければならない
成功だけでなく、
拒否・失敗も制度の一部である
Change 1 — event_type 定義（最小）

Change 2 — 記録責任の定義
Receiver  → rejection を生成
Executor  → failure を生成
Recorder  → completed を生成
Change 3 — ATOS側の受け取り
すべてのexecution系イベントは
research.trace_event に記録される
Change 4 — 必須記録原則
Rule 1: すべてのAABは最終的に
        いずれかの結果を持つ

Rule 2: 未記録状態は禁止

Rule 3: success / reject / fail
        のいずれか1つに必ず分類される
Change 5 — aab_bundle_id の役割強化
すべてのexecutionイベントは
aab_bundle_idで追跡可能であること
明示的スコープ外
❌ 詳細なresultスキーマ標準化 → 将来
❌ 実行ログの長期保存戦略 → 将来
❌ 外部DBとの同期 → 将来
❌ 分散AEEA統合 → 将来
