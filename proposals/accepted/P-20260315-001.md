## Background

Founder policy statement (2026-03-14):

> Do not over-complicate documentation. Write what new AI instances
> need to know and respect in soft language. Full memorization is not
> required. Separate knowledge by role and specialization.

This proposal implements that policy through structural reorganization
of the ATOS document system.

## Core Design: Three-Layer Knowledge Model

```
Layer 0 — All AiiD (common, minimal)
Layer 1 — Librarian / Proposer (governance)
Layer 2 — Collector / Executer (technical specialization)
```

---

## Change 1 — constitution.md Restructuring

**Problem**
constitution.md contained both the core constitution (Articles 1–10)
and enacted amendments (Articles A–G) in a single file, implicitly
requiring all participants to read the full document.

**Change**
Separate the amendment texts into individual files.

```
constitution.md              ← Articles 1–10 only (principles)
docs/amendments/
  FR-20260312-003.md         ← Execution Authority Clarification
  FR-20260312-004.md         ← AI Identity and Continuity
```

constitution.md now ends with an amendment index:

```
For enacted amendments, see: docs/amendments/

All amendments remain constitutionally enacted.
This restructuring only relocates amendment texts for clarity.
```

**Important**
This is a relocation, not a deletion. All amendments remain fully
enacted and in effect.

## Change 2 — Anchor Directory Restructuring

**Problem**
All anchors were stored flat in docs/anchors/, making role-specific
navigation difficult and unsustainable as the project grows.

**Change**

```
docs/anchors/
  ATOS_BOOTSTRAP_ANCHOR.md        ← All AiiD (entry point)
  AI_TRADING_OS_MASTER_ANCHOR.md  ← All AiiD (overview)

  governance/
    EXECUTION_MODEL_ANCHOR.md     ← Librarian / Proposer

  technical/
    DB_STATUS_ANCHOR.md           ← Collector / Executer
    OPENCLAW_REGISTRAR_TRAINING_ANCHOR.md ← Executer
```

All AiiD read only the top two files. Role-specific anchors are
referenced as needed.

**Bootstrap sequence unchanged**
The bootstrap read order remains unchanged. This restructuring affects
document organization, not the conceptual reconstruction sequence.
