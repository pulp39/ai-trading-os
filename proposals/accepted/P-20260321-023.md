4. Conceptual Layer
4.1 Executionの定義（再確認）
ExecutionはGateの延長ではない
Executionは別の制度行為である
4.2 状態遷移
preview → gate_passed → execution（未定義）
                ↓
           blocked（本Proposalで定義）
4.3 核となる原則
executionは「通る」より先に「止められる」べき

6. Blocked Path Layer
6.1 Block条件（必須）

以下のいずれかで必ず停止：

B-1_preview_missing
B-2_gate_preview_mismatch
B-3_preview_not_found
B-4_real_order_not_authorized
B-5_scope_ambiguous
B-6_role_violation
B-7_duplicate_execution_context
B-8_gate_task_mismatch
6.2 条件定義詳細
B-1_preview_missing
linked_preview_event_id が未指定
B-2_gate_preview_mismatch
gate_event と preview_event の不整合
B-3_preview_not_found
preview_event がDBに存在しない
B-4_real_order_not_authorized
task.metadata.real_order_authorized ≠ true
B-5_scope_ambiguous
scope が不明確または未定義
B-6_role_violation
role boundary違反
B-7_duplicate_execution_context
同一コンテキストの重複実行
B-8_gate_task_mismatch
gate_event.real_order_authorized ≠ task.real_order_authorized
6.3 最重要Block条件
B-4_real_order_not_authorized

👉 execution_gate_passed 単体では execution は発生しない

8. Governance Layer
8.1 なぜ「止める」が重要か
実行能力より停止能力が優先される
8.2 停止の制度的意味
停止は失敗ではない
停止は制度的成功である

execution_blocked は execution_gate_passed と同等の制度的重みを持つ
8.3 Gateとの関係
Gateは通過可能状態を作る
Executionは別の判断である
