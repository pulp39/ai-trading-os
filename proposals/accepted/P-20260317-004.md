Background
P-20260317-003によりindicator_observationが制度化されました。しかし現状では：
indicator_observation → （誰も読まない）→ proposal未生成
という断絶が存在します。本Proposalはこの断絶を埋め、ATOSに自己駆動ループを与えます。
Observation → Interpretation → Hypothesis → Proposal
この経路が閉じた瞬間、ATOSは「記録するシステム」から「思考するシステム」へ移行します。

ATOS 4層モデル（完成形）
Observation Layer    : board_snapshots
Execution Layer      : collector_run
Interpretation Layer : indicator_observation
Hypothesis Layer     : hypothesis_proposed  ← 本Proposalで新設

Change 1 — Hypothesis Triggerの制度的定義
Hypothesis Triggerは以下として定義します：
Hypothesis Trigger is a deterministic rule mapping
indicator_observation sequences to hypothesis
generation conditions.
重要： Triggerは「構造」を定義します。具体的数値は実装時に決定されますが、制度として以下の構造を固定します：
trigger_type ∈ {
  "threshold"   : |value| が閾値を超えた場合
  "consecutive" : 同一銘柄でN連続同一interpretationの場合
}

Change 2 — ProposerはObservation Inputを「受ける」
Librarianの修正を採用します：
（修正前）Proposerは trace_event を読む
（修正後）Proposerは Observation Input を受ける制度主体
indicator_observation
↓
Input Stream（trigger condition評価）
↓
Proposer（hypothesis生成）
これは将来の自動化・外部AI接続における供給モデルの基盤となります。

Change 3 — hypothesis_proposed event_typeの定義
hypothesis_proposedをresearch.trace_eventの正式なevent_typeとして定義します。現段階では新テーブル不要。
metadata：
json{
  "event_type": "hypothesis_proposed",
  "proposed_by": "Proposer",
  "trigger_type": "consecutive",
  "trigger_condition": "3 consecutive bullish",
  "symbol": "7203",
  "indicator_id": "vwap_deviation",
  "hypothesis": "7203 shows sustained buying pressure above VWAP",
  "confidence": "low",
  "basis_event_ids": [156, 157, 158],
  "aab_bundle_id": null
}
