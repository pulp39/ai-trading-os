Background
P-20260319-006によりElevation Layerが成立しました。しかし現状では：
formalize_trading_proposal.py
→ 引数ベース（--title, --summary）
→ reviewを確認しない
→ review未通過のdraftでもTP生成が可能
これは制度破壊リスクに該当します。本Proposalはこれを閉じます。
「人間が正しく使う」
↓
「間違っても壊れない」

Core Concept — Elevation Integrity Layer
「何を正式化するか」（draft参照自動化）
＋
「それを正式化してよいか」（reviewゲート）
= Elevation Integrity Layer
Input:
--draft-event-id 206
Validation:
1. proposal_reviewed が存在するか？
2. review_result = "approved" か？
3. 最新のreviewを参照（ORDER BY id DESC LIMIT 1）
Resolution:
draft metadata を自動取得
Output:
trading_proposal（TP）

Change 1 — draft metadata 自動取得（修正③反映）
修正後（After）：
bashpython formalize_trading_proposal.py \
  --draft-event-id 206
内部で以下を自動取得：
sqlSELECT metadata
FROM research.trace_event
WHERE event_type = 'proposal_drafted'
  AND id = 206
LIMIT 1;
```

取得するフィールド：
```
title
summary
symbol
source_hypothesis_event_id
confidence
agent_id
triggered_at

Change 2 — Review Gate Enforcement（修正①②反映）
formalizeの実行前に以下を必須チェックとします：
sqlSELECT *
FROM research.trace_event
WHERE event_type = 'proposal_reviewed'
  AND metadata->>'source_proposal_draft_event_id' = '206'
  AND metadata->>'review_result' = 'approved'
ORDER BY id DESC
LIMIT 1;
```

**修正内容：**
```
修正① metadata->>'result' → metadata->>'review_result'
修正② ORDER BY id DESC LIMIT 1（最新reviewを参照）
条件未達の場合：
pythonraise ElevationIntegrityError(
  "Cannot formalize: review not approved for draft_event_id=206"
)
```

→ TPは生成されません。
```
