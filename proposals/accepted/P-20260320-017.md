Proposal P-20260320-017 Rev.1
Title: Order Preview and Price Binding Layer
Date: 2026-03-20
From: Proposer (Claude)
Type: Execution Preparation Layer
Constitutional Impact: None
Scope: simulated_order → real_order間の価格拘束・preview記録

Background
simulated_orderの開通により、次のギャップが明確になりました：
simulated_order：estimated_price = null
real_order     ：具体的な価格拘束が必要

→ その間に「価格を観測に基づいて固定する」
  レイヤーが必要

Core Concept（修正版）
Order Preview = 「実行前に観測値を記録する行為」
estimated_price = Collector寄り（観測の延長）
readiness = execution gateの前段条件フラグ（記録のみ）

Change 1 — estimated_priceの定義（①反映）
estimated_price
  = board_snapshots.current_price の参照値
  = 観測の延長（解釈なし）
  = 単一snapshot_idにバインド（②反映）

計算・推定は行わない
直近のsnapshot 1件のcurrent_priceをそのまま使用

Change 2 — order_preview_recorded event_type（⑤反映）
trace_event拡張として定義：
event_type = "order_preview_recorded"

```
---

### Change 3 — readinessの定義（③反映）
```
readiness ∈ {"ready", "not_ready", "stale"}

ready     : execution gateに進める状態
not_ready : 条件未達（snapshot古い等）
stale     : 有効期限切れ

制度上の意味：
  ✅ execution gateの前段条件フラグ
  ❌ 実行許可ではない
  ❌ proposal強化ではない
  現段階では記録のみ・実行には影響しない
```

---

### Change 4 — slippageの除外（④反映）
```
slippageはpreview層のスコープ外

理由：
  実注文後にのみ確定する値
  preview段階での定義は
  execution未実装での実行仮定の導入になる

→ 将来のexecution層（P-018以降）で定義
```

---

### Change 5 — 再現可能性の定義（⑥反映）
```
reproducible = true の意味：
  同一source_snapshot_idを参照すれば
  同じpreview結果が再現可能

deterministic（完全決定性）は要求しない
市場データの性質上、完全決定性は保証不可能
```

---

### Change 6 — 明示的スコープ外
```
❌ slippage定義         → P-018以降
❌ real_order実装       → P-018以降
❌ 複数snapshot集約     → Indicator層の責務
