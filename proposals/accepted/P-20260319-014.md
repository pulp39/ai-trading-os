Proposal P-20260319-014
Title: Capital Allocation — Execution Constraint Definition
Date: 2026-03-19
From: Proposer (Claude)
Type: Execution Constraint Layer — Capital Definition
Constitutional Impact: None
Scope: AAB拡張フィールドとしての資本制約定義（実売買未接続）

Background
P-20260319-013でExecution Recordingが確立されました。ATOSは「何が実行されたか」を記録できます。しかし現状では：
TP：何をやるか
AAB：どの範囲で許可するか
Capital Allocation：どの強度で許可するか ← 未定義
本Proposalはこの「強度」を制度として定義します。

Core Concept（Librarian整理に基づく）
Capital Allocation
= 実行条件の一部（constraint）
= 資本移動そのものではない

位置づけ：
TP → AAB（Capital Allocation含む）→ TMP → OpenClaw → execution_recorded
ATOSは実行しない原則を維持します。

Change 1 — Capital Allocationの制度的定義
Capital Allocation = AABに含まれる実行制約パラメータ

定義のタイミング：AAB生成時（事前）
更新のタイミング：execution_recordedを踏まえた次TP生成時

Change 2 — AABへの拡張フィールド
既存のAAB構造に以下を追加します：
json{
  "bundle_type": "AAB",
  "capital_allocation": {
    "enabled": true,
    "currency": "JPY",
    "max_amount": 100000,
    "max_position_ratio": 0.01,
    "risk_level": "minimal",
    "constraint_type": "observation_only"
  }
}
```

**フィールド定義：**
```
max_amount        : 最大投入金額（円）
max_position_ratio: 総資産に対する最大比率
risk_level        : minimal / low / medium / high
constraint_type   : observation_only / order_permitted

Change 3 — 現段階の制約値
現テスト環境では以下を強制します：
json{
  "capital_allocation": {
    "enabled": false,
    "constraint_type": "observation_only"
  }
}
```
```
enabled = false    → 資本制約は定義のみ（未適用）
constraint_type    → observation_onlyを維持
```

実売買への接続は将来フェーズで別途Proposalを起草します。

---

### Change 4 — 更新サイクル
```
第1回execution_recorded
  ↓ Proposerが結果を参照
  ↓ 次のTPで資本制約を再定義
  ↓ 新AABに反映
第2回execution_recorded
  ↓ ...
```

これは既存のObservation → Hypothesis → Proposalループに自然に統合されます。

```
