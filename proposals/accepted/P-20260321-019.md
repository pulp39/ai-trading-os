Proposal P-20260321-019
Title: Order Preview Off-Hours Test Override
Date: 2026-03-21
From: Proposer (Claude)
Type: Test Governance — Off-Hours Override Definition
Constitutional Impact: None
Scope: bounded/dry-run/preview_only条件下でのoff-hours override定義

Background
P-20260320-017によりorder_previewが開通しました。しかし現行実装では：
市場時間外 + stale snapshot
→ readiness = stale / not_ready
→ ready系フローの検証が市場時間に強く依存
本Proposalはbounded条件下でoff-hours overrideを制度化し、ready系フローの検証を時間に依存せず実施できるようにします。

Core Principle
off-hours override は：
  ✅ 検証目的のみ
  ✅ bounded / dry-run / preview_only条件下に限定
  ❌ 実市場条件の代替ではない
  ❌ execution authority を与えない
  ❌ readiness の制度的意味を変更しない

Change 1 — off-hours override 適用条件
以下の全条件を満たす場合のみ適用可能：
① dry_run = true
② preview_only = true
③ AABに "test_override_allowed": true が明示されている
④ task artifactに "off_hours_test_override": true が明示されている
⑤ external_order_allowed = false
⑥ state_change_allowed = false
いずれか1つでも欠ける場合は通常の判定ルールを適用します。

Change 2 — readiness判定の修正（override適用時のみ）
通常時：
market_closed    → readiness = stale / not_ready
snapshot_stale   → readiness = stale / not_ready
override適用時：
market_closed    → failure扱いから除外（記録は維持）
snapshot_stale   → failure扱いから除外（記録は維持）
→ readiness = ready（他条件クリア時）
重要： overrideはreadiness判定ロジックの一時的緩和であり、制度的な意味を変更しません。

Change 3 — 記録要件（必須）
override適用時は以下フィールドを必須記録：
json{
  "event_type": "order_preview_recorded",
  "execution_subtype": "order_preview",
  "test_mode": true,
  "off_hours_test_applied": true,
  "stale_snapshot_test_override": true,
  "readiness": "ready",
  "failed_checks": [],
  "overridden_checks": ["market_closed", "snapshot_stale"],
  "dry_run": true,
  "external_order_sent": false,
  "reproducible": true
}
overridden_checksフィールドにより何が緩和されたかが追跡可能になります。

Change 4 — task artifact拡張
registrar_queue/order_preview_task.jsonに以下を追加：
json{
  "off_hours_test_override": true,
  "override_conditions": {
    "market_closed_allowed": true,
    "stale_snapshot_allowed": true,
    "requires_dry_run": true,
    "requires_preview_only": true
  }
}

Change 5 — AAB拡張
off-hours overrideを使用するAABには以下が必要：
json{
  "constraints": {
    "dry_run": true,
    "preview_only": true,
    "external_order_allowed": false,
    "state_change_allowed": false,
    "test_override_allowed": true
  }
}
```

### Change 6 — 互換性制約（明示）
```
✅ 既存simulated_orderの変更なし
✅ 既存order_preview（通常）の変更なし
❌ real_order未導入
❌ external order禁止
❌ capital mutation禁止
❌ execution authority付与なし
❌ readinessの制度的意味の変更なし
```

---

### Change 7 — 明示的非目標
```
❌ 実市場条件の代替ではない
❌ 市場時間外での実売買の許可ではない
❌ overrideを通常運用に昇格させない
❌ readiness=readyが実行を意味するわけではない
```

### Impact Summary

| Item | Change |

| `overridden_checks` | 緩和された条件を追跡 |

| Constitutional impact | None |
```
