Core Concept
market_observation → simulated_order → real_order
（段階的移行の制度的根拠）

simulated_order ≠ real_order
  実際の注文を出さない
  市場への影響なし
  注文ロジックの検証のみ

Change 1 — simulated_orderの定義
simulated_order：
  定義：実注文を出さずに注文ロジックを
        シミュレートする実行パス

  制約：
    ✅ 注文ロジックの実行
    ✅ 注文パラメータの生成・記録
    ❌ 実際の注文送信
    ❌ 証券口座への接続
    ❌ 資本移動

```
---

### Change 3 — OpenClaw側開通条件
```
① Receiver側：
   action_type = "simulated_order" を
   有効なパスとして認識する

② Executor側：
   dry_run = true の場合、
   注文パラメータを生成するが
   送信しない（dry_run実行）

③ Recorder側：
   execution_recorded +
   execution_subtype = simulated_order
   として記録する

Change 4 — event_type定義（修正①反映）
既存のexecution_*系に統一します：
json{
  "event_type": "execution_recorded",
  "execution_subtype": "simulated_order",
  "aab_id": "AAB-20260319-004",
  "target_agent_id": "openclaw",
  "status": "simulated",
  "order_params": {
    "symbol": "8306",
    "order_type": "market",
    "quantity": 100,
    "side": "buy",
    "estimated_price": 2750.0
  },
  "dry_run": true,
  "external_order_sent": false,
  "recorded_by": "OpenClaw/Recorder",
  "recorded_at": "2026-03-19T10:15:01+09:00"

```

**修正内容：**
```
廃止：simulated_order_recorded
採用：execution_recorded + execution_subtype=simulated_order
理由：event_type爆発防止・DB互換性・クエリ整合
```

---

### Change 5 — rejection境界の明文化
```
① dry_run = false → 拒否
② external_order_allowed = true → 拒否
③ simulated_orderパスが未実装 → 拒否
④ constraint_type が simulated_only 以外 → 拒否
```

```
