Background
P-20260317-001・002により、ATOSは観測・保存・実行記録の三層を確立しました。本Proposalは最初のIndicatorを定義し、ATOSを「記録するシステム」から「意味を生成する制度」へ移行させます。
またFounderの構想に基づき、本Proposalから以下を制度として前提とします：
ATOS = Trust-Oriented Interpretation System
  → 実行しない
  → 「何を信頼するか」を定義し、伝達する

Core Concept
board_snapshots（状態）
↓ Indicator計算
indicator_observation（意味）
↓ Proposerへの入力
proposal（仮説）

ATOS 正式4層モデル（本Proposalで確定）
Observation Layer    : board_snapshots
Execution Layer      : collector_run
Interpretation Layer : indicator_observation  ← 本Proposalで新設
Hypothesis Layer     : proposal

Change 1 — Collectorの制度的位置づけ
Collector = Authorized External Execution Agent (AEEA)

ATOSの内部プロセスではない
外部AIプロセスとして制度的に承認されている
承認根拠：activity_authorization(openclaw) → 1（FR-20260316-001）
ATOSは承認ハッシュを付与し、外部AIへ伝達する
実行はCollectorが行い、ATOSは実行しない


Change 2 — Indicator: vwap_deviation の正式定義
indicator_id : vwap_deviation
version      : 1.0
source_table : public.board_snapshots
fields_used  : current_price, vwap
計算式：
value = (current_price - vwap) / vwap
解釈（制度定義）：
interpretation ∈ {"bullish", "neutral", "bearish"}

value > 0 : bullish  （買い優勢）
value = 0 : neutral
value < 0 : bearish  （売り優勢）
拡張可能性：
intraday analysis → mean reversion → breakout detection
→ AAB自動判断トリガーへ接続

Change 3 — indicator_observation event_type の定義
indicator_observationをresearch.trace_eventの正式なevent_typeとして定義します。

P-20260317-003 establishes the first interpretation layer of ATOS by defining VWAP deviation as an institutional indicator. It aligns with the AAB future model by treating collectors as authorized external execution agents and ATOS as a trust-definition system rather than an execution engine.
